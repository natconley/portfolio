Infinite Oddities

Node.js PostgreSQL REST API Repository Pattern Express

About the project

Infinite Oddities is a full-stack inventory management system for a fictional curiosity shop. Workers can add, edit and delete products and suppliers through a clean interface backed by a relational database.

What I built

Everything, this was a solo project. REST API with Node.js and Express, a PostgreSQL database with relational tables for products and suppliers, and a vanilla JS frontend that talks to it all. The file structure follows the Repository Pattern, separating database logic, business logic and routes so each layer has one clear responsibility.

Challenges

I had a solid conceptual understanding of how backend systems work before this project. What I didn't have was the feeling of it. Getting the frontend to actually talk to the backend, watching data move from a browser form all the way into a database and back was the moment everything stopped being abstract and started being real.

What I learned

PostgreSQL, RESTful API design, and how to structure a backend so it doesn't become a mess. But mostly I learned that understanding something and building something are two very different things, and that the second one is where it actually sticks. I also learned that sometimes, starting over is easier and quicker than going back.

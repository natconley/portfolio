React Student Project

Pokédex - Pokémon Team Builder

DOM Manipulation PokéApi Vanilla JS Responsive Design Accessibility

About the project

A collaborative Pokédex pulling live data from the PokéAPI, built during a JavaScript 1 course. The focus was on dynamic websites and API integration — no frameworks, just vanilla JS and the DOM.

What I built

I owned the teambuilder page end to end, UI and logic. Add up to six Pokémon to your team and get a breakdown of strengths, weaknesses, type coverage and stat balance, plus suggestions for Pokémon that could improve your team. Team and waitlist data persists in localStorage so you don't lose your squad on refresh. I also put deliberate focus on accessibility, making sure dynamic content updates and interactions worked for keyboard navigation and screen readers, not just mouse users.

Challenges

Without component-based thinking, the page grew complex fast. renderSuggestions, renderTeam and renderWaitlist each built their own cards from scratch, the same logic repeated three times. It worked, but it wasn't pretty.

What I learned

This project taught me a lot about what I'd do differently. As a non-pokémon player myself, it was made clear how important user research can be. The analysis logic was genuinely interesting to figure out, but looking back the type coverage could give the user so much more, specific matchup breakdowns rather than just a summary, and knowing more about what a user wants to figure out would've been a huge benefit.

I'd also pull the card rendering into a single reusable function. That instinct for reusability is exactly what clicked properly when I started working with React components. Overall the code could use a good restructuring.

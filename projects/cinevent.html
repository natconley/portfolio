React Student Project

Cinevent

React Zustand OMDB API TMDB API Responsive Design

About the project

Cinema isn't dead, at least not if you're going to Cinevent. A fictional cinema selling an extraordinary film experience, built as a group React project.

What I built

I owned the data pipeline from API to UI. Setting up the OMDB and TMDB API calls, building the Zustand stores for movies and favorites, and creating the components that tie it together. The details page, MovieCard, favorites button, favorites page and all movies page were all made by me.

What I learned

This project was my proper introduction to React. JSX clicked quickly, but components really started to make sense when I had to decide what deserved to be its own component and what didn't. MovieCard and putting pages together taught me that.

Zustand was a revelation after passing props around. Building the movies and favorites stores made me understand why shared state exists. Once the data lives in one place, everything that needs it just reaches for it.

React Router gave the app structure that felt real. Navigating between pages without a reload is one of those things that sounds simple until actual set up.

I also got more comfortable with hooks than I expected. useState for local UI state, useEffect for fetching on mount. Knowing which one to reach for, and when, was the thing that took the most time to feel natural.
